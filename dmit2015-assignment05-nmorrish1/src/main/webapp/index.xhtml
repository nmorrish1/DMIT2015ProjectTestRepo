Discussion

For this assignment, I have completed the following tasks:

  • All Web application URLs are transported over HTTPS
  • Intranet (LDAP) users can create, edit, and list their own events
  • Anonymous users can view calendar events for all users, but cannot edit events without logging in.
  • Logged in users can logout.
  • After 5 failed login attempts, users are redirected to https://www.nait.ca/programs/dmit-computer-software-development?term=2020-winter
  • Authenticated LDAP Users have their name displayed (in the top right)
  • Authenticated LDAP users can have reminders sent to their account email
  • Custom error pages for HTTP status 401, 403, 404, and 500

Known issues:

  • Clicking on urls to Create/Edit when initially logged out will redirect users to the login page. After successfully logging in, the Create/Edit page does not display user name in the top right as it should.
